### Types of Contributions Welcome

1. **Bug Fixes** - Found an issue? Help us fix it!
2. **Feature Enhancements** - Better ways to demonstrate existing features
3. **New Features** - Additional Rust stdlib features to showcase
4. **Documentation Improvements** - Better explanations, examples, or organization
5. **Code Quality** - Better practices, cleaner code, improved structure
6. **Translation** - Help translate documentation to other languages

### Getting Started

1. **Fork the repository** on GitHub
2. **Clone your fork** locally
3. **Create a new branch** for your changes
4. **Make your changes** and test them
5. **Submit a pull request** with a clear description

### Development Setup

```bash
# Clone the repository
git clone https://github.com/yourusername/rust-stdlib-showcase.git
cd rust-stdlib-showcase

# Create a feature branch
git checkout -b your-feature-name

# Build and test
cargo build
cargo run
```

### Code Style Guidelines

- **Follow Rust conventions** - Use `rustfmt` for formatting
- **Clear naming** - Use descriptive variable and function names
- **Good comments** - Explain complex logic, not obvious code
- **Error handling** - Use proper Rust error handling patterns
- **Safety first** - Avoid `unsafe` unless absolutely necessary

### Adding New Features

If you want to add a new Rust stdlib feature:

1. **Check if it's already included** - Review existing features
2. **Keep it educational** - Focus on clarity over complexity
3. **Add to the main showcase** - Integrate with existing structure
4. **Update documentation** - Add to README and comments
5. **Test thoroughly** - Ensure it works on different platforms

### Pull Request Process

1. **Describe your changes** clearly in the PR description
2. **Reference any related issues** if applicable
3. **Keep PRs focused** - One feature/fix per PR when possible
4. **Be responsive** to feedback and questions
5. **Update documentation** if your changes affect usage

### Code of Conduct

This project follows the [Rust Code of Conduct](https://www.rust-lang.org/policies/code-of-conduct). Please be respectful and inclusive in all interactions.
